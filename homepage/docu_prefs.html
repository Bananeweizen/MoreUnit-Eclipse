Preferences

To use MoreUnit you can/should configure the preference page to fit your personal needs. All these settings can be configured for the whole workspace or for each project.
  • Directory for testcases: The directory for your testcase should be the folder where you have your tests.


    In the follwoing screenshot the source for tests are stored in the JUNIT-folder:


    If you store your test in a different project you have to use the project specific settings and configure the testproject:


  • Testcase suffixes/prefixes: Please configure the prefix/suffix-lists to your preferred naming convention:
  • Package suffixes/prefixes: If you are using prefixes / suffixes for your tests you have to enter the prefxi / suffix here.
    For example: If your test for classes in the package com.foo.* are stored unter com.foo.test.* you have to enter "test" into the package suffix property.
  • Create JUnit4 methods: If you enable this option, moreUnit generates JUnit4 testmethods.
  • You can configure a super class which is used in the new test case wizard. You need to enable JUnit4 or TestNG otherwise org.junit.TestCase is used as super class.
  • Enable flexible naming of tests: If you are having several testcases for each special situation you should active this option.
    Example:
    Classname: Foo.java
    Tests: FooTest.java, FooSituation1Test.java, FooSituation2Test.java
    Just enter "Test" as your testcase suffix name in MoreUnit, activate "enable flexible naming of tests" and MoreUnit will find all tests for Foo.java. Using the jump to test action MoreUnit shows a light-weight to let the developer choose which is the test he wants to jump to.
