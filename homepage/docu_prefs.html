Preferences

To use MoreUnit you can/should configure the preference page to fit your personal needs. All these settings can be configured for the whole workspace or for each project.
  • Directory for test cases: The directory for your test case should be the folder where you have your tests.


    In the following screenshot the source for tests are stored in the JUNIT-folder:


    If you store your test in a different project you have to use the project specific settings and configure the test project:


  • Use test-prefix for test-methods: Automatically checked when using JUnit3:
  • Method content: The default content of test methods generated by MoreUnit:
  • Test case suffixes/prefixes: Please configure the prefix/suffix-lists to your preferred naming convention:
  • Package suffixes/prefixes: If you are using prefixes / suffixes for your tests you have to enter the prefix / suffix here.
    For example: If your test for classes in the package com.foo.* are stored under com.foo.test.* you have to enter "test" into the package suffix property.
  • You can configure a super class which is used in the new test case wizard. You need to enable JUnit4 or TestNG otherwise org.junit.TestCase is used as super class.
  • Enable flexible naming of tests: If you are having several test cases for each special situation you should active this option:


    Example:
    Class: Foo.java
    Tests: FooTest.java, FooSituation1Test.java, FooSituation2Test.java
    Just enter "Test" as your test case suffix name in MoreUnit, activate "enable flexible naming of tests" and MoreUnit will find all tests for Foo.java. Using the jump to test action MoreUnit shows a light-weight to let the developer choose which is the test he wants to jump to.
